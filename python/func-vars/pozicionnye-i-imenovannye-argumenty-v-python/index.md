## Введение

Аргументы функций позволяют передавать данные внутрь функций для обработки. В Python существуют позиционные и именованные аргументы, что даёт гибкость вызова функций и улучшает читаемость кода.

## Позиционные аргументы

Позиционные аргументы передаются функции в порядке их определения:

```python
def greet(name, age):
    print(f"Hello, {name}, you are {age} years old")

greet("Alice", 30)  # Hello, Alice, you are 30 years old
```

Порядок имеет значение: `greet(30, "Alice")` вызовет некорректный результат.

## Именованные аргументы

Именованные аргументы указываются явно по имени при вызове:

```python
greet(age=30, name="Alice")  # Hello, Alice, you are 30 years old
```

Именованные аргументы можно комбинировать с позиционными:

```python
greet("Alice", age=30)  # Hello, Alice, you are 30 years old
```

## Аргументы по умолчанию

Часто удобно задавать значения по умолчанию:

```python
def greet(name, age=25):
    print(f"Hello, {name}, you are {age} years old")

greet("Bob")  # Hello, Bob, you are 25 years old
greet("Bob", 40)  # Hello, Bob, you are 40 years old
```

## Использование аргументов в выражениях

```python
def add(a, b=5):
    return a + b

print(add(3))      # 8, использовано значение по умолчанию
print(add(3, 7))   # 10, указано новое значение
```

## Частые ошибки

* Перепутан порядок позиционных аргументов.
* Использование позиционных аргументов после именованных → `SyntaxError`.
* Неявное смешивание именованных и позиционных аргументов без учета порядка.

## Часто задаваемые вопросы

1. **Можно ли использовать только именованные аргументы?**
   Да, при вызове функции можно передавать все аргументы по имени.

2. **Можно ли изменить значение аргумента по умолчанию во время вызова?**
   Да, достаточно указать новое значение при вызове.

3. **Что происходит, если аргумент с дефолтным значением не указан?**
   Используется значение по умолчанию.
